What breaks with OpenClaw memory — verbatim from users
Pain 01

MEMORY.md overflows within a week

It grows. You prune it. It grows again. Within days it's too big to load efficiently, half of it is stale, and the agent has no structured way to know what's relevant. You end up maintaining a second job: curating your agent's memory.

"Overflows within a week of heavy usage. Requires constant manual pruning and the agent has no structured way to know what's in it." — GitHub issue #51612, filed this week

Pain 02

Sessions reset. Everything forgotten.

After 60 minutes idle, you're a stranger. Everything you taught the agent — preferences, context, your projects — gone. Unless you remembered to say "write this to MEMORY.md." Which you forgot. Because that's not how people work.

"I went to bed feeling like I'd finally cracked it. Next morning: 'I'm sorry, I don't have any recollection of that.' Hours of work. Vanished." — r/openclaw, 2 weeks ago

Pain 03

Every message burns tokens on stale context

The default fix is dumping your entire memory file into every prompt. That's 6K+ tokens of stale context on every single message — slow, expensive, and the agent still gets confused by contradictions in old data.

"The fixed cost overhead of all the SOUL, MEMORY etc is pretty big." — OpenClaw Discord, #general, this week

Pain 04

Memory wiped after an update

You upgrade OpenClaw. Next morning the agent remembers nothing. The files still exist but the agent can't use them. This has happened to thousands of users and it's happened more than once.

"After upgrading, the next morning it didn't remember anything, although memory files existed and SQLite was in place." — r/openclaw, 2 weeks ago

Pain 05

Semantic search requires a separate API key that fails silently

OpenViking requires its own embeddings setup. Most users skip it. Memory search silently degrades to keyword matching. The agent "feels like it started over" when it can't find relevant context.

"Memory search needs an embeddings provider key. Codex OAuth doesn't cover embeddings. This is a huge gotcha." — OpenClaw support bot, repeated 20+ times

How Known works

After every session ends, Known automatically extracts what matters — preferences, patterns, facts — into a local knowledge graph. No "write this to MEMORY.md" required.

Semantic search
